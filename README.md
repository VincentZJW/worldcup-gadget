# 中文

## WorldCup Gadget

世界杯桌面悬浮球。下载 macOS App 后，桌面会显示一个小足球悬浮球；点击小足球展开最新战报，再次收起回到桌面悬浮状态。

### 下载使用

访问下载页：

[https://vincentzjw.github.io/worldcup-gadget/](https://vincentzjw.github.io/worldcup-gadget/)

下载页会跳转到 GitHub Releases。下载最新的 macOS `.dmg` 后，把 `WorldCup Gadget.app` 拖到 `Applications`，然后打开应用即可。

### 你会看到什么

- 一个 macOS 桌面小足球悬浮球。
- 点击悬浮球展开世界杯战报 dashboard。
- 展示最新比分、国家旗帜、进球球员、进球时间和比赛摘要。
- 顶部“刷新”会尝试更新远程战报 feed，然后读取本地缓存。
- 应用会自动从远程 JSON feed 更新数据，用户不需要安装 Codex、OpenClaw 或任何开发工具。

### 首次打开提示

当前免费分发版本是 unsigned beta，还没有接入 Apple Developer ID 签名和公证。macOS 首次打开时可能提示“无法验证开发者”或阻止直接打开。

这是未签名 macOS App 的正常限制，不代表应用需要付费服务。请只从本仓库的 GitHub Releases 下载。

### 数据更新

桌面 App 运行时会从远程 JSON feed 更新战报，并写入本机应用数据目录中的缓存。界面渲染时读取本地缓存，所以短暂网络失败时仍能显示上一次成功更新的数据。

远程 feed 由 GitHub Actions 自动生成：

- `.github/workflows/update-worldcup-feed.yml` 定时抓取 FIFA 官方数据并更新 `data/latest.json`。
- `floating-gadget/main.js` 默认读取 jsDelivr CDN feed，并在失败时回退到备用地址。
- 应用里的“刷新”按钮会触发一次远程更新，再重新读取本地缓存。

### 免费分发方式

本项目当前采用：

- GitHub Pages：产品下载页。
- GitHub Releases：存放 macOS `.dmg` 安装包。
- GitHub Actions：自动部署下载页、手动打包并创建 Release、定时更新比赛数据 feed。

这种方式可以免费推广和测试，但体验不等同于已签名公证的正式 macOS App。正式大规模分发前，仍建议后续接入 Apple Developer ID 签名和 notarization。

### 维护发布

维护者可以在 GitHub Actions 手动运行 `Build Mac Release` workflow：

1. 输入 release tag，例如 `v1.0.0-beta.1`。
2. 等待 workflow 在 macOS runner 上构建 unsigned DMG。
3. workflow 会创建或更新同名 GitHub Release，并上传 `.dmg`。
4. GitHub Pages 下载页会始终引导用户访问 latest release。

GitHub Pages 由 `Deploy GitHub Pages` workflow 自动部署 `docs/` 目录。首次启用时，需要在 GitHub 仓库 `Settings > Pages` 中选择 GitHub Actions 作为发布来源。

### 项目结构

- `floating-gadget/`：Electron 桌面悬浮球 App。
- `data/latest.json`：首次启动 seed data，也是远程 feed 的 committed JSON。
- `scripts/generate_worldcup_feed.mjs`：生成世界杯战报 feed。
- `docs/`：GitHub Pages 下载页。
- `.github/workflows/`：数据更新、下载页部署和 macOS Release 打包 workflow。

# English

## WorldCup Gadget

WorldCup Gadget is a macOS desktop floating soccer-ball app for following the latest World Cup match report. After installing the Mac app, a small floating ball stays on the desktop. Click it to open the match dashboard, then collapse it back into the desktop gadget.

### Download

Visit the download page:

[https://vincentzjw.github.io/worldcup-gadget/](https://vincentzjw.github.io/worldcup-gadget/)

The download page links to GitHub Releases. Download the latest macOS `.dmg`, drag `WorldCup Gadget.app` into `Applications`, and open the app.

### What You Get

- A small macOS desktop floating soccer ball.
- Click the floating ball to open the World Cup match dashboard.
- Latest scores, flags, scorers, goal minutes, and match summaries.
- The top Refresh button updates the remote report feed first, then reloads the local cache.
- Match data updates automatically from a remote JSON feed. Users do not need Codex, OpenClaw, or any developer tools.

### First Launch Note

The current free distribution is an unsigned beta. It does not yet use Apple Developer ID signing or notarization. On first launch, macOS may warn that the developer cannot be verified or block the app from opening directly.

That is a normal limitation of unsigned macOS apps. Please download only from this repository's GitHub Releases.

### Data Updates

The desktop app updates match reports from a remote JSON feed and writes them into the local app data cache. The UI renders from that local cache, so it can keep showing the last successful update if the network temporarily fails.

The remote feed is generated by GitHub Actions:

- `.github/workflows/update-worldcup-feed.yml` periodically fetches FIFA data and updates `data/latest.json`.
- `floating-gadget/main.js` reads the default jsDelivr CDN feed and falls back to backup URLs if needed.
- The app's Refresh button triggers a remote update, then reloads the local cache.

### Free Distribution

This project currently uses:

- GitHub Pages for the product download page.
- GitHub Releases for the macOS `.dmg` installer.
- GitHub Actions to deploy the page, build release assets manually, and update the match feed on a schedule.

This is enough for free beta testing and lightweight promotion, but it is not the same experience as a signed and notarized macOS app. For broad public distribution, Apple Developer ID signing and notarization are still recommended.

### Maintainer Release Flow

Maintainers can manually run the `Build Mac Release` workflow in GitHub Actions:

1. Enter a release tag, for example `v1.0.0-beta.1`.
2. Wait for the workflow to build the unsigned DMG on a macOS runner.
3. The workflow creates or updates the matching GitHub Release and uploads the `.dmg`.
4. The GitHub Pages download page always sends users to the latest release.

GitHub Pages is deployed from `docs/` by the `Deploy GitHub Pages` workflow. On first setup, choose GitHub Actions as the publishing source in `Settings > Pages`.

### Project Structure

- `floating-gadget/`: Electron desktop floating gadget app.
- `data/latest.json`: first-launch seed data and the committed remote feed JSON.
- `scripts/generate_worldcup_feed.mjs`: World Cup report feed generator.
- `docs/`: GitHub Pages download site.
- `.github/workflows/`: data update, Pages deployment, and macOS Release workflows.
